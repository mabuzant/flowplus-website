# flow+ SEO / AEO + Auto-Blog Engine ‚Äî Design

**Date:** 2026-07-22
**Canonical domain:** https://flowplus.ae
**Repo:** mabuzant/flowplus-website (branch `main`, Railway auto-deploy)
**Stack:** static HTML (`index.html`, `founder.html`) + Express `server.js`, bilingual EN/AR.

## Goal

Give flow+ the technical + content foundation to compete for top rankings on Google
and Bing, and to be cited by answer engines (Claude, Gemini, ChatGPT, Perplexity), plus
a self-sustaining weekly blog that keeps the site fresh. No ranking is *guaranteed*; the
work maximizes odds through clean crawlable markup, structured data, AEO-friendly Q&A
content, and a fresh-content engine.

**Two audiences (both):**
- Regional: MENA/UAE AI-workshop & training seekers ("AI workshop Dubai", "creative AI training UAE").
- Global: creative-technology / AI-studio clients ("creative technology company", "AI creative studio").

**Reddit angle:** target the real question phrasing people use on Reddit inside FAQ +
blog content (no live embeds), so flow+ becomes the lifted answer in LLM conversations.

## Non-goals

- No paid API keys ‚Äî all automation runs through the Claude subscription CLI (headless).
- No external CMS. Blog is file-based static HTML.
- No live Reddit embedding or auto-posting to Reddit.
- No guarantee of a specific rank position.

## Workstream 1 ‚Äî Technical SEO foundation

Applies to `index.html` and `founder.html`, plus new server routes and files.

**Meta layer (per page):** unique `<title>`, `meta description`, `link rel=canonical`,
`meta robots` (index,follow), Open Graph (`og:title/description/image/url/type/locale`
with `ar_AR` alternate), Twitter Card (`summary_large_image`). OG image: a new
flow+-branded OG asset (`assets/flowplus-og.png`, 1200√ó630) to be created during build.

**`robots.txt`** (new, served at `/robots.txt`): allow all; `Sitemap: https://flowplus.ae/sitemap.xml`;
explicitly welcome `GPTBot`, `ClaudeBot`, `Google-Extended`, `PerplexityBot`, `Bingbot`, `Applebot-Extended`.

**`sitemap.xml`** (new, served at `/sitemap.xml`): home, founder, `/blog`, every post;
`lastmod`, `changefreq`, `priority`. Regenerated by the blog engine on each new post.

**JSON-LD structured data** (in `<head>`):
- `Organization` ‚Äî name flow+, `url`, `logo`, `sameAs` = [`https://instagram.com/itsflowplus`, `https://www.linkedin.com/company/flow-plus-ae/`], `areaServed` MENA, `founder` ‚Üí Person.
- `WebSite` ‚Äî with `potentialAction` SearchAction.
- `Course` / `Event` for workshops (drawn from existing workshop data where available).

**Server (`server.js`):** add routes for `/robots.txt`, `/sitemap.xml`, `/blog`,
`/blog/:slug`; correct `Content-Type` headers; keep the existing SPA-style catch-all but
ensure blog + SEO files take precedence over it.

**hreflang:** self-referential `alternate` links for EN/AR since language is toggled client-side; `html lang` remains `en` with AR handled via toggle.

**Crawl hygiene:** exactly one `<h1>` per page, descriptive `alt` on images, heading order audit, lazy-load audit.

## Workstream 2 ‚Äî On-page + AEO

- **Homepage AEO block:** a concise, quotable FAQ ("What is flow+?", "What does flow+ do?",
  "Where is flow+ based?", "How do flow+ AI workshops work?") in the direct factual style
  LLMs quote verbatim, marked up with `FAQPage` JSON-LD.
- **Reddit-style phrasing:** FAQ + posts phrased like real user questions ("is an AI workshop
  worth it", "how do agencies actually use AI").
- **Entity clarity:** consistent NAP, founder as `Person` linked from Organization, `sameAs` socials.

## Workstream 3 ‚Äî Blog system (`/blog`)

- File-based: `blog/posts/<slug>.html` rendered from a shared brand template (Space Grotesk /
  Cairo, dark theme, matches `css/flow-plus.css`).
- `/blog` index page lists posts from a `blog/posts.json` manifest (title, slug, date, excerpt, tags, lang).
- Per post: `BlogPosting` + `FAQPage` JSON-LD, OG tags, canonical, breadcrumb schema, prev/next.
- **Seed "hidden" post:** fully live, indexable, in sitemap, **not** linked from nav ‚Äî URL shared directly. Launches the cadence.
- `posts.json` is the single source of truth driving index, sitemap regeneration, and "recent posts".

## Workstream 4 ‚Äî Weekly auto-writing engine

Location: `.claude/blog-engine/` in the repo.

- **`topics.json`** ‚Äî backlog of ~20 SEO/AEO-researched titles covering both audiences, each with target keyword + intended reader question. Consumed top-down; the engine marks used topics.
- **`write-next-post.md`** ‚Äî a self-contained prompt/workflow the Claude CLI runs headless. On each run it:
  1. Picks the next unused topic from `topics.json`.
  2. Writes a full brand-voice post (EN, with AR-ready structure) + all schema.
  3. Emits `blog/posts/<slug>.html`, appends to `posts.json`, marks the topic used.
  4. Regenerates `sitemap.xml`.
  5. `git add -A && git commit && git push` ‚Üí Railway auto-redeploys.
- **`run.sh`** ‚Äî wrapper that invokes `claude` headless against `write-next-post.md` from the repo root and handles logging.

## Workstream 5 ‚Äî Local scheduler

- A weekly recurring task **on the user's laptop** (via `scheduled-tasks` MCP, or a `launchd`
  plist / cron entry) that executes `.claude/blog-engine/run.sh`.
- Runs locally as specified; no cloud dependency. Logs to `.claude/blog-engine/logs/`.

## Dependencies / risks to verify during build

- Working `git push` credentials on the laptop and Railway auto-deploy on `main` (required for auto-publish).
- Social profiles for `sameAs`: Instagram `@itsflowplus`, LinkedIn `flow-plus-ae` (confirmed).
- OG image: new flow+-branded `assets/flowplus-og.png` to be created (confirmed).
- Headless `claude` CLI available in PATH for the scheduler.

## Success criteria

- All pages validate: Rich Results Test passes for Organization/WebSite/FAQ/BlogPosting.
- `robots.txt` + `sitemap.xml` reachable and correct at the canonical domain.
- Seed post live, indexable, in sitemap, not in nav.
- Running `run.sh` once produces a committed, deployed new post end-to-end.
- Weekly local schedule fires the engine unattended.
